# UI Medium-Term Ticket Structure Guide

This directory breaks `plans/ui-roadmap-medium-term.md` into standalone tickets for future sessions.

The same frontmatter conventions used in `plans/tickets/` apply here:

- `title`
- `status`
- `type`
- `priority`
- `blockers`
- `sources`
- `related_notes`

Suggested ticket types for this directory:

- `ui-feature`
- `workflow-policy`
- `future-design`
- `docs`

Medium-term UI tickets should usually focus on deeper verification workflow, mixed-initiative suggestion, reusable working sets, graph-quality maintenance, and mode-specific behavior while preserving explicit user control over context.
